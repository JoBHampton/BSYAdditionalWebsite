Application Features

  • Account system

    Users create an account and sign in to access saved recipes and itineraries. All data is tied to the user's authenticated profile.

  • Recipe management

    Users add new recipes, edit existing ones, and delete anything they no longer need. All recipe data is stored and retrieved from the backend.

  • Recipe selection

    Users select one or multiple recipes and move them into a checkout flow. The system compiles all chosen items into a single plan.

  • Cost estimation

    The application aggregates ingredients across the selected recipes and outputs an estimated total cost.

  • Printable ingredient list

    All ingredients are merged, deduplicated, and displayed as a clean list formatted for printing.

  • Itinerary creation

    Users save groups of selected recipes as named itineraries for later reuse. Itineraries can be loaded and used in new sessions.

  • Dietary filters

    The interface includes preset dietary filters (Vegetarian, Vegan, GF, Peanut-Free, Treenut-Free, Soy-Free, Lactose-Free, Diabetes, Kosher). Users filter recipes instantly with no page reload.

  • Favorite recipes

    Users can mark recipes as favorites and view them in a dedicated section on the dashboard.

  • Recipe detail modal

    Clicking a recipe opens a modal displaying full instructions, servings, tags, and an “Add to Cart” button.

  • Cart and checkout persistence

    Selected recipes stay loaded during navigation. Users can move between pages without losing their recipe selections.

  • Serving-size scaling

    Checkout lets users set the number of people. Ingredient quantities scale automatically based on the chosen group size.

  • Live price lookup

    The system fetches estimated prices for each ingredient and displays a line-item breakdown plus a total cost.

  • PDF export

    Users download the entire shopping list as a formatted PDF with one click.

  • Automatic itinerary creation

    Each checkout generates a timestamped itinerary by default, which users can rename and reuse.

  • Recipe tags and metadata

    Every recipe card displays dietary tags, estimated prep time, and base serving size.

  • Dashboard navigation

    Users move between dashboard, recipe database, and checkout with persistent navigation buttons.
