How To Make Modifications

  • Assuming someone has followed the instruction for installing the application, how can they make changes?

    All modifications can be made by editing the source code. If you want to add a new schema to the database, add the sechema to the backend folder in the bsyBackend repository, an entry point to the api folder, and source in vercel.json.

  • What compiler is used?

    The website uses the Vite build tool.

  • What languages are used?

    The majority of the code is in JS/JSX

  • Where are dependencies listed?

    All dependencies are on the install page. All necessary npm commands to run the website locally are listed.

  • Build management and automated builds:

    All source code is provided in the project GitHub. Currently there is not a testing environment for the website, nor is there any build automation.

  • Project backlog:

    There are two issues on the project backlog. A generative AI API needs to be implemented for suggesting recipe modifications. Editing functionality for itineraries needs to be added.

  • Project bug list:

    There are currently no known bugs.

  • Testing and test cases:

    There are no test cases for this website.

  • Style expectations:

    A color scheme and set of icons have been chosen and implemented for this project. Any future work should maintain the same scheme and iconography.
